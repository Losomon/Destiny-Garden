# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Destiny Gardens, please report it responsibly.

### Supported Versions

- `main` branch: ✅ Actively supported
- Older tags/releases: ❌ No longer supported

### Reporting Guidelines

Submit vulnerability reports via:
- **Email**: hello@destinygardens.co.ke (preferred for sensitive issues)
- **GitHub Issues** (public): Label as `security` if non-sensitive

Include:
- Description of vulnerability
- Steps to reproduce (minimal)
- Impact assessment
- Proposed fix (optional)

### What We Do
- Acknowledge receipt within 48 hours
- Assess internally within 7 days
- Release fix/patch as needed
- Credit reporters in changelog (opt-out available)

### Common Issues Handled
- XSS/CSRF in booking form
- Form validation bypass
- Schema injection
- Performance DoS

### Disallowed
- Doxing or public shaming
- Competing project attacks

We follow [GitHub's security guidelines](https://docs.github.com/en/code-security).

Thanks for helping keep Destiny Gardens secure! 🔒
